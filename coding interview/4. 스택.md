## 4. 스택
```
스택
데이터 삽입과 삭제가 한쪽 끝에서 이루어지는 순서가 있는 특별한 형태의 리스트.
제일 먼저 추가된 데이터가 제일 마지막에 나간다. (FILO)
제일 마지막에 추가된 데이터가 제일 먼저 나간다. (LIFO)

참고로 자바의 stack은 기본적으로 synchronization이 적용되어있다.
이 부분을 인지하고 사용하자 / synchronization이 없는 ArrayDeque를 대용으로 사용할수도있다.
```
```
주요 매서드 시간 복잡도
push : O(1)
pop : O(1)
peek/top : O(1)

stack을 배열과 리스트 둘 다로 구현해볼 수 있는데 시간복잡도는 동일하다
아래 자바 배열 & 리스트로 구현한 스택을 보면서 이해하자
```

## stack - array로 구현
```
class MyStack {
    int size;
    int[] arr;
    int topIndex;

    public MyStack(int size) {
        this.size = size;
        this.arr = new int[size];
        this.topIndex = -1;
    }

    public void push(int val) {
        if (topIndex == size - 1) {
            return;
        }

        arr[++topIndex] = val;
    }

    public int pop() {
        if (topIndex == -1) {
            return 'x';
        }

        return arr[topIndex--];
    }

    public int peek() {
        if (topIndex == -1) {
            return 'x';
        }

        return arr[topIndex];
    }
}
```
## stack - list로 구현
```
class Node {
    int val;
    Node prev;
    Node next;

    public Node(int val) {
        this.val = val;
        this.prev = null;
        this.next = null;
    }
}
class MyStack {
    Node head;

    public MyStack() {
        this.head = null;
    }

    public void push(Node node) {
        node.next = head;
        head = node;
    }

    public Node pop() {
        if (head == null) {
            return null;
        }
        Node target = head;
        head = head.next;
        return target;
    }

    public Node peek() {
        if (head == null) {
            return null;
        }

        return head;
    }
}
```
