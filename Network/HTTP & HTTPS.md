### HTTP & HTTPS

### HTTP
```
hypertext transfer protocol
- 인터넷상에서 클라이언트와 서버가 자원을 주고받을 때 사용하는 통신 규약

★http는 텍스트의 교환
-> 누군가 네트워크에서 신호를 가로채면 내용이 노출되는 보안 이슈가 존재
-> 이 문제를 해결하는 프로토콜이 https
```

### HTTPS
```
hypertext transfer protocol secure
- 인터넷상에서 정보를 암호화하는 SSL 프로토콜을 사용해 클라이언트와 서버가 자원을 주고 받을 때 사용하는 통신규약
```

### HTTPS의 통신 흐름
```
1. 서버는 https 적용을 위해 공개키와 개인키/ 비대칭키를 만든다.
2. 신뢰할 수 있는 CA(certificate authority)인증 기관을 선택하고 해당 기업에게 서버 공개키 관리를 부탁
3. 클라이언트가 서버에게 ex.html파일을 달라고 요청
4. 해당 요청은 Https요청이 아니기 때문에 ca가 서버 공개키 전달
5. 클라이언트의 브라우저는 서버 공개키로 요청을 암호화하여 전달
6. 서버는 개인키로 요청을 복호화
```
