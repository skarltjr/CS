### 운영체제가 여러 프로그램을 동시에 실행하는 원리가 무엇인가요?

```
답변 : 
바로 cpu scheduling

cpu scheduling이란 cpu가 보다 많은 일을 수행하며 빠른 응답시간을 도출하기위한 프로세스 배정 방법이다

프로세스가 종료될때까지 기다리는 비선점 방식과
cpu를 강제로 회수하는 선점방식이 존재한다

비선점 방식에 대표적인 예시는 선입선출 or 짧은 수행시간 작업 우선처리 방식이 있다.
비선점방식은 cpu를 강제로 회수할 수 없기 때문에 문제가 발생할 수 있다.
예를들어 먼저 들어온 순서대로 처리하게 된다면 수행시간이 매우 긴 프로세스가 먼저 들어오면 다음에 들어온
짧은 프로세스는 가벼운 작업임에도 많은 시간 대기


선점 방식에는 라운드로빈 / multilevel-feedback-queue등이 존재한다

라운드로빈은 모든 프로세스에게 동일한 시간을 분배하여 각 프로세스마다 ex)10초 수행 후 cpu가 회수된다
그러나 이 경우 무거운 프로세스는 긴 시간이 필요한 경우가 발생하는데
따라서 등장한것이 multilevel-feedback-queue

multilevel-feedback-queue는 일단 ex)10초를 분배받고 만약 끝내지못했다면 다음 
레벨(우선순위가 낮아진다) 큐로 이동시키는 작업 반복
우선순위가 낮아진 큐는 다음에 cpu를 할당받을 때 상위 레벨보다 좀 더 긴 시간을 분배받는다.

이러한 cpu scheduling을 통해 여러 프로세스가 실행된다.

```
