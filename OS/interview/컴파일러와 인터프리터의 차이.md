### 컴파일러와 인터프리터의 차이는 무엇인가요?
```
공통점 :

컴파일러와 인터프리터 모두 프로그래밍 언어를 기계어로 변환한다.
```

```
차이점 : 그 과정에서 차이가 있다.


컴파일러 : 
- 소스코드 전체를 훑고 명령어를 수집하여 컴퓨터 프로세서가 실행할 수 있도록 바로 기계어로 변환
- 소스코드 -> 기계어
- 전체 코드를 기계어로 변환 후 에러를 보고
- 전체적인 관점에서 인터프리터보다 빠르다.

인터프리터:
- 소스코드 각 행을 연속적으로 분석하며 실행
- 인터프리터는 소스코드 -> 중간형태 -> 기계어
- 소스코드 각 행을 변환하다 에러를 반환하면 그 후 코드는 변환하지 않고 멈춘다.(보안관점 우수)
```


```
자바는 컴파일러와 인터프리터 병행


컴파일러를 통해 .java파일을 기계어인 .class 파일로변환(컴퓨터가 아닌 jvm을 위한 변환)
⭐️그리고 이 .class파일이 어떤 플랫폼에서든 동작할 수 있게(jvm은 os에 종속적이지 않다) 어떤 컴퓨터에서든 동작할 수 있게
변환하기 위해서 인터프리터가 사용된다

```
