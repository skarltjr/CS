### 프로세스와 스레드
```
프로세스 : 메모리 상에서 실행중인 프로그램 작업
스레드 : 프로세스 안에서 실행되는 여러 흐름 단위

기본적으로 하나의 프로세스는 하나의 스레드를 갖고있다.
```

### 프로세스
```
프로세스는 각각 별도의 주소공간을 할당받는다.(독립적)


Code : 코드 자체를 구성하는 메모리 영역(프로그램 명령)
Data : 전역변수, 정적변수, 배열 등
초기화 된 데이터는 data 영역에 저장
초기화 되지 않은 데이터는 bss 영역에 저장

Heap : 동적 할당 시 사용 (new(), malloc() 등)

Stack : 지역변수, 매개변수, 리턴 값 (임시 메모리 영역)
```

### 스레드
```
프로세스와 다르게 스레드는 스레드간 스택 영역만 할당받고 나머지는 스레드간 공유
```

### 차이
```
즉 프로세스는 별도의 주소 공간을 할당받아 독립적 동작, 다른 프로세스에게 주소 공간 침범 x

스레드는 스택을 제외한 나머지를 스레드간 공유
```

### 멀티 프로세스
```
여러 cpu를 통해 여러 프로세스를 병렬,동시에 처리
장점 : 안정성 / 프로세스간 독립적인 자원,주소공간 할당으로 침범 x
단점 : 각자 독립된 주소공간(메모리)를 사용하기때문에 오버헤드 발생가능(컨텍스트 스위칭시)

context switching이란?
- 프로세스의 상태 정보를 저장하고 복원하는 일련의 과정
- 여러 프로세스가 실행될 때 b프로세스 실행 후 -> a 실행 후 -> 다시 b를 수행할 때 이전까지 작업내용/상태를 복원하기 위한 행위
- 멀티 프로세스의 경우 프로세스 단위로 동작하며 프로세스는 독립적인 메모리 영역(code,data,stack,heap)을 할당받기에 해당 작업에서 오버헤드가 크다 
```

### 멀티 스레드
```
하나의 프로세스를 여러 스레드를 통해 작업
장점 : 스레드는 code,data,heap 영역을 공유하고 있기에 스레드 동시 수행 시 context switching과 같은 오버헤드가 적다
단점 : 안정성 / 말 그대로 stack을 제외한 나머지 영역을 공유하고 있기 때문에 하나의 스레드가 해당 메모리 영역을 망가뜨리면 모든 스레드가 고장
```
